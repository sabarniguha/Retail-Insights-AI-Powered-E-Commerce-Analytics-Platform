# 📊 Retail Insights — AI-Powered E-Commerce Sales Analysis

A production-grade Business Intelligence dashboard built with Streamlit, Plotly, and Scikit-learn — styled and structured like an internal analytics tool at a company such as Amazon, Flipkart, Shopify, or Walmart.

## ✨ Features

| Page | What it does |
|---|---|
| 🏠 **Dashboard** | KPI cards (revenue, profit, orders, customers, AOV, margin, quantity), monthly trends, top categories/products/regions, auto-generated insights |
| 📈 **Sales Analytics** | Daily/weekly/monthly/yearly views, revenue & profit trends, discount analysis, shipping analysis, day-of-week × month heatmap |
| 🛍 **Product Analytics** | Best/worst sellers, category treemap, revenue/profit breakdowns, full product table |
| 👥 **Customer Analytics** | RFM metrics, repeat customer rate, CLV, **K-Means customer segmentation** with cluster scatter plot and segment profiles |
| 🌍 **Regional Analytics** | Region/state performance, **choropleth map**, **bubble map**, state performance table |
| 📦 **Inventory Insights** | Automated fast/slow-mover detection (90-day velocity), low performers, high-demand products, restocking recommendations |
| 📉 **Sales Forecasting** | Random Forest / XGBoost / Linear Regression forecasts for sales or orders, actual-vs-predicted backtesting, MAE/RMSE/R² |
| 🤖 **AI Business Assistant** | Ask natural-language questions; answered by Groq or Mistral (auto-switches on failure), with an always-available offline rule-based fallback grounded in your live data |
| 📊 **Reports** | Export filtered data as CSV, multi-sheet Excel, or a PDF executive summary |
| ⚙ **Settings** | Theme, AI provider/model, forecast horizon, currency, date format |

All charts use **Plotly** with a consistent dark theme. All pages respect **global sidebar filters** (date range, category, sub-category, region, state).

## 🗂 About the Dataset

`data/ecommerce_sales_data.csv` contains ~9,850 order-line records (2021–2024) generated by `generate_data.py`.

> **Note on data provenance:** this project was built in an offline sandbox with no access to third-party data hosts, so a live proprietary retail export couldn't be downloaded. Instead, `generate_data.py` produces a large, internally consistent synthetic dataset that mirrors the real-world structure of the well-known "Global Superstore" e-commerce schema — realistic seasonality (Nov/Dec peaks), category-level margin economics, repeat-customer (Pareto) behavior, regional distributions, and shipping logistics. Every chart, KPI, ML model, and forecast in the app operates on this CSV exactly as it would on a real retailer's export — nothing is hard-coded.
>
> **To use your own data:** replace `data/ecommerce_sales_data.csv` with any CSV containing at minimum these columns: `Order ID, Order Date, Ship Date, Customer ID, Customer Name, Segment, City, State, Region, Category, Sub-Category, Product Name, Sales, Quantity, Discount, Profit`. The app validates the schema on load and shows a friendly error if columns are missing.

To regenerate the sample dataset:
```bash
python generate_data.py
```

## 🚀 Quick Start

```bash
# 1. Clone / download this project, then install dependencies
pip install -r requirements.txt

# 2. (Optional) enable the AI assistant with real LLM answers
cp .env.example .env
# then edit .env and add GROQ_API_KEY and/or MISTRAL_API_KEY

# 3. Run the app
streamlit run app.py
```

The app works fully **without** any API keys — the AI Business Assistant automatically falls back to a built-in, data-grounded rule-based engine.

## ☁️ Deploy to Streamlit Community Cloud

1. Push this project to a public (or private) GitHub repository.
2. Go to [share.streamlit.io](https://share.streamlit.io) and click **New app**.
3. Point it at your repo, branch, and `app.py`.
4. (Optional) Under **Advanced settings → Secrets**, add:
   ```toml
   GROQ_API_KEY = "your-key"
   MISTRAL_API_KEY = "your-key"
   ```
5. Click **Deploy**. That's it — `requirements.txt` handles all dependencies.

## 🧱 Tech Stack

- **Frontend:** Streamlit
- **Data processing:** Pandas, NumPy
- **Visualization:** Plotly (Express + Graph Objects) — line, area, bar, pie, treemap, choropleth, bubble map, heatmap, scatter
- **Machine learning:** Scikit-learn (K-Means, Random Forest, StandardScaler, evaluation metrics), XGBoost (optional forecasting model)
- **AI Assistant:** Groq / Mistral LLM APIs with automatic provider fallback, plus an offline rule-based engine
- **Reports:** OpenPyXL (Excel), fpdf2 (PDF)
- **Config:** python-dotenv

## 🏗 Architecture

Everything lives in a single, modular `app.py` (~1,600 lines) organized into clear sections:

```
app.py
├── Configuration & global styling (inject_css, kpi_card, format_currency)
├── Data layer (load_data, apply_filters, compute_kpis) — all @st.cache_data
├── Sidebar navigation + global filters (render_sidebar)
├── Business insights engine (generate_business_insights) — rule-based, always available
├── Page 1-10 render functions (page_dashboard, page_sales_analytics, ...)
├── ML helpers (build_customer_features, run_customer_segmentation,
│               prepare_forecast_dataset, train_forecast_model, forecast_future)
├── AI assistant (build_data_context_summary, call_groq, call_mistral,
│                 rule_based_fallback_answer)
├── Report generators (generate_pdf_report, generate_excel_report)
└── main() — routing + top-level error handling
```

### Performance
- `st.cache_data` on data loading, KPI computation, feature engineering, and forecast dataset prep.
- Heavy computations (clustering, forecasting) only re-run when filters or parameters change.
- Sampling used for dense scatter plots to keep rendering fast on large filtered sets.

### Error handling
- Missing/invalid/empty CSV → friendly on-screen message, no crash.
- Missing AI API keys → automatic fallback to the offline insights engine.
- Any unexpected page-level exception is caught in `main()` so one broken page never takes down the whole app.
